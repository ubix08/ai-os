# Blog Patterns & Templates

## Standard Article Structure
```markdown
---
title: "Title"
description: "Meta description (150-160 chars)"
date: YYYY-MM-DD
lastUpdated: YYYY-MM-DD
tags: [tag1, tag2]
author: "Full Name"
---

> Key Takeaways
> - Bullet 1
> - Bullet 2
> - Bullet 3

Hook paragraph (1-2 sentences that grab attention).

## Section 1
Answer-first opening sentence. Context and explanation. Code examples where relevant.

## Section 2
... (continue with H2 sections)

## FAQ
### Q1?
Answer in 1-2 sentences.

### Q2?
Answer in 1-2 sentences.
```

## Article Variants

### Tutorial
- Prerequisites section
- Step-by-step with numbered lists
- Expected output after each step
- Troubleshooting at the end

### Comparison
- Feature comparison table
- Use case scenarios for each option
- Verdict with recommendation criteria

### Deep-Dive
- Architecture diagram (placeholder)
- Layer-by-layer breakdown
- Performance benchmarks
- Trade-off analysis

### Listicle
- Numbered items with detailed explanations
- Each item: what it is, why it matters, how to use it
- Summary table at the end

### Changelog / Release Notes
- Version and date in frontmatter
- Categories: Added, Changed, Deprecated, Removed, Fixed, Security
- Each entry links to relevant docs/PRs

### Case Study
- Problem statement
- Solution architecture
- Implementation journey
- Results with metrics
- Lessons learned

## SEO Elements
- Title formula: "Primary Keyword: Compelling Benefit" or "How to [Action] in [Tool]"
- Internal links to 3-5 related articles
- External links to 3-5 unique authority domains
- Images with descriptive alt text and captions

## File Naming
Format: `YYYY-MM-DD-slug.md`
Directory: `content/articles/`
