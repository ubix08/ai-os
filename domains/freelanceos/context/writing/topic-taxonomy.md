# Topic Taxonomy

## Topic Categories

### Infrastructure & DevOps
- Default audience: Intermediate
- Voice: Direct, operational, reliability-focused
- Code emphasis: YAML, bash, Dockerfile, Terraform, CI config
- Key signals: deployment patterns, scalability, monitoring

### APIs & Backend
- Default audience: Intermediate
- Voice: Precise, implementation-focused
- Code emphasis: Python, TypeScript, Go, cURL examples, OpenAPI specs
- Key signals: endpoints, authentication, error handling, performance

### Frontend & UI
- Default audience: Intermediate
- Voice: Practical, UX-aware
- Code emphasis: TypeScript, React/Vue/Svelte, CSS, HTML
- Key signals: component architecture, state management, accessibility

### AI & ML
- Default audience: Beginner-Intermediate
- Voice: Explanatory, trend-aware
- Code emphasis: Python, Jupyter, CLI tools, API usage
- Key signals: model selection, prompt engineering, evaluation metrics

### Security & Compliance
- Default audience: Intermediate-Advanced
- Voice: Authoritative, cautionary
- Code emphasis: Python, bash, config files, audit tools
- Key signals: threat models, encryption, compliance frameworks

### Developer Experience & Tooling
- Default audience: All levels
- Voice: Enthusiastic, practical
- Code emphasis: CLI examples, config files, editor/IDE setup
- Key signals: productivity, automation, workflow improvement

## Audience Skill Levels

| Level | Assumes | Avoids |
|-------|---------|--------|
| Beginner | Basic programming concepts | Framework-specific knowledge, advanced patterns |
| Intermediate (default) | Domain fundamentals | Niche or cutting-edge topics |
| Advanced | Deep domain expertise | Basic explanations, introductory context |

## Calibration Rules
- Default to Intermediate unless the topic demands otherwise
- If the topic spans categories, use the highest default audience
- For ambiguous topics, include audience note in the article metadata
- When in doubt, optimize for the most experienced reader without leaving intermediates behind
